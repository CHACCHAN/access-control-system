# 顔登録仕様

## 概要

顔認証パネルの「顔を登録する」から、任意のメンバーに現在カメラに写っている顔の
特徴ベクトル(embedding)を登録する。登録はサーバー(顔特徴ベクトル登録 API)へ
保存され、次回以降のメンバー取得時に照合対象として配布される。

## フロー

1. 登録オーバーレイ(`FaceRegistrationOverlay`)を開き、メンバーを検索・選択する。
2. 「この顔を登録する」を押すと Rust 側の `capture_face_embedding` を実行:
   - 最新フレームから顔検出(SCRFD)
   - 顔が検出できない → エラー「顔が検出できませんでした」
   - 顔幅がフレーム幅の `minFaceWidthRatio` 未満 → エラー「顔が小さすぎます」
   - 106点ランドマークでアライメント → ArcFace で 512次元 embedding を抽出
3. 抽出した embedding を顔特徴ベクトル登録 API へ POST する:
   - `POST {postEndpoint}/{username}`
   - ボディは設定のテンプレート(既定 `{"descriptor": "{{descriptor}}"}`)から生成
   - `Authorization` ヘッダーに設定の API トークンをそのまま付与
4. 成功したらセッション内の照合対象(`enroll`)へ即時反映し、Rust 側へも同期する。
   端末の再起動やメンバー再取得後はサーバー配布値が使われる。

## タイムアウト・自動クローズ

- 登録オーバーレイは操作がないまま 60 秒(`IDLE_TIMEOUT_MS`)経過すると
  自動的に閉じて認証モードへ戻る(キオスクの放置対策)。

## エラーハンドリング

- 抽出エラー(顔なし・小さすぎ)は Rust 側から日本語メッセージ付きで返り、
  そのままオーバーレイに表示される。
- API 登録失敗時は「登録に失敗しました: …」を表示し、照合対象へは反映しない。
